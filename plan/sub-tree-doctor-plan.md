# サブ樹木医システム 構築計画

## Context（なぜ作るか）

依頼者は樹木医。NotebookLMノートブック「樹木医必携」（ID: `0eb78ec8-fc95-470d-b43b-d64a4dc6a809`）には、樹木医の実務に不可欠な2冊が入っている：

1. **木材腐朽菌図鑑**（`26b7e1ec`）— 主要65種の腐朽菌をカラー写真で網羅。樹種・腐朽部位・子実体形態から菌を絞り込む**簡易検索表**、診断法・対策付き。
2. **樹木医の手引き 第4版**（`74118ac3`）— 生理生態・構造の基礎から病害虫・環境ストレス・法制度・倫理、**EBD（科学的根拠に基づく診断）**までの総合技術指導書。

目的は、この2冊を土台に依頼者（樹木医）の診断・業務を支援する「サブ樹木医システム」を構築すること。ユーザー選択により**統合システム（3層構成）**で作る。

## 抽出済みの土台知識（出典: 上記2冊, `notebooklm ask`で取得済み）

- **EBD標準診断フロー**：①予備調査/概況 → ②外観診断（樹勢判定・VTA・病徴標徴・木槌打診）→ ③精密診断（穿孔抵抗レジストグラフ・応力波PiCUS・γ線・生長錐/フラクトメーター・エアースコップ根系）→ ④危険度判定（構造安定性・t/R比）→ ⑤総合判定・処方箋・モニタリング
- **腐朽菌 簡易検索表の分類軸（5系統）**：宿主（針葉樹/広葉樹）／発生部位（根株・地際/幹・枝）／子実体の持続性・質感（一年生・多年生／軟・硬）／外部形態（有柄・無柄、傘表面）／子実層托（ヒダ/管孔/針状）＋組織色・腐朽型（白色/褐色）
- **重要腐朽菌10種**：コフキタケ、ベッコウタケ（最重要警戒）、カワラタケ、カワウソタケ、シマサルノコシカケ、ナラタケ、マツオウジ（褐色）、アイカワタケ（褐色）、カイメンタケ（褐色）、ツリガネタケ 等（宿主・部位・腐朽型・危険度特徴つき）

## 成果物（3層）

### ① 対話診断プロトコル（即使用可）
- ベテラン樹木医（サブ樹木医）として振る舞う運用プロトコルを定義：
  - 依頼者の入力（樹種・部位・症状・子実体・環境）を受け、EBDフローの各段に沿って、
  - `notebooklm ask -c e22fbe00...`（既存会話）で2冊を根拠に照会し、
  - **出典付き**で候補菌・推奨精密診断法・危険度評価・対策を返す。
- プロトコル本体は Markdown で明文化し、③でNotebookLMのノートにも保存する。

### ② HTML診断支援ツール（オフライン単体動作 / Artifact）
- **artifact-design スキルを読んでから**作成。テーマ対応・レスポンシブ・自己完結（CSP準拠）。
- 機能：
  1. **腐朽菌 簡易検索ナビ**：宿主(針/広葉)→発生部位→子実体持続性・質感→子実層托→腐朽型 の選択で候補菌を絞り込み表示（重要10種＋主要種を内蔵データ化）。各菌に宿主/部位/腐朽型/危険度メモ。
  2. **EBD診断フロー チェックリスト**：①〜⑤の段階を進行管理。各段の観察項目・使用機器を提示。
  3. **危険度スコア（t/R比）計算**：健全材厚 t と半径 R を入力→倒木危険度の目安を提示（あくまで参考・出典注記）。
  4. **精密診断機器リファレンス**：レジストグラフ/PiCUS/γ線/生長錐/フラクトメーター/エアースコップの用途早見。
- データは②内に静的JSとして保持（本HTMLはNotebookLMへ問い合わせない＝現地オフライン参照可）。
- 医学的免責・出典（2冊）を明記。

### ③ NotebookLM体系化成果物（ノートブック内）
`notebooklm generate` で2冊を根拠に生成（生成は確認のうえ実行）：
- **診断プロトコル レポート**（`generate report --format custom` に①のプロトコルを反映）
- **腐朽菌クイックリファレンス**（`generate data-table` で 菌名/宿主/部位/腐朽型/危険度 の表）
- **樹木医学 マインドマップ**（`generate mind-map`、即時・低リスク）
- （任意）**EBD学習スタディガイド**（`generate report --format study-guide`）
- 生成は rate limit の可能性があるため、mind-map→data-table→report の順で堅い方から。失敗時は artifact list 確認のうえ再試行 or スキップ。
