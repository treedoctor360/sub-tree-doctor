# サブ樹木医システム

樹木医の実務を補助する **対話診断アプリ**（React + Vite / GitHub Pages）。
NotebookLM「樹木医必携」の2冊（**樹木医の手引き 第4版** ／ **緑化樹木腐朽病害ハンドブック**）を
根拠に、Gemini を頭脳として **共に考え・的確な指示を出し・診断結論へ導き・記録に残す**、
ベテラン先輩樹木医（サブ樹木医）。

> **原則：AIは候補出し・整理・観察ガイドの助手。診断と処置の最終判断・責任は樹木医が負う。**

## エコシステム内の役割
- **shindanv27**：診断項目の入力（健全度14・活力度17）と総合判定・所見・地図。
- **wood-decay-fungi**：腐朽菌の同定（写真AI＋65種）。
- **サブ樹木医（本アプリ）**：上記の診断結果を受け、2冊を根拠に対話で推論し、次の一手・
  精密診断の指示・危険度（t/R）の見立て・処置方針を導き、相談記録として残す**頭脳層**。

## 機能
- **対話**：Gemini（GAS透過中継を流用）を頭脳に、知識ベースでグラウンディングした先輩樹木医との対話。
- **カルテ**：shindanv27 記録のJSON取り込み、健全度14／活力度17、総合判定の自動計算、
  「樹木医の手引き」由来の追加項目（立地・土壌診断／精密診断の実測値／危険度・標的／腐朽／病虫害／処置）。
- **記録**：対話とカルテから診断レポートを生成→編集→保存（IndexedDB＋記録GASで共有スプレッドシートへ同期）。
- **参照**：t/R危険度計算・精密診断機器・主要腐朽菌クイックリファレンス。

## セットアップ
1. 依存関係：`npm install`
2. 開発：`npm run dev` ／ ビルド：`npm run build`
3. **記録GAS**（任意・共有する場合）：`gas/diagnosis-db-relay.gs` を GAS にデプロイし、
   スクリプトプロパティ（SHEET_ID / TOKEN / SHEET_NAME）を設定。発行URLとトークンをアプリの「設定」に登録。
4. **Geminiリレー**（各自で用意）：利用者ごとに `gas/gemini-relay.gs` を自分の GAS プロジェクトにデプロイし、
   スクリプトプロパティ（GEMINI_API_KEY / TOKEN、RAGを使うなら CORPUS_FILE_ID）を設定。ウェブアプリとして
   「全員」に公開し、その `/exec` URL と TOKEN をアプリの「設定」に登録する（費用は各自の Google アカウントに課金）。
   公開ビルドには特定の GAS URL を埋め込まない（既定は空）。
5. **公開**：main へ push すると GitHub Actions が `dist` を Pages へデプロイ
   （Settings → Pages → Source を「GitHub Actions」にしておく）。

## 技術
React 18 / Vite 5 / IndexedDB / GAS→Gemini（gemini-2.5-flash 透過中継）。外部送信は
Geminiリレー（対話）と記録GAS（同期）のみ。設定値は端末内（localStorage）に保存。
